Highlight #1 About Version Control

Version control is a system that records changes to a file or project so that one can recall a file from an earlier point. This is incredibly useful because if a mistake is made, it is possible to easily restore progress to the point of failure. The two types of version control systems (VCSs) are centralized and distributed. Centralized version control systems (CVCSs) have a server that contains all the versioned files while distributed version control systems (DVCSs) fully mirror the repository, so if one server gets shut down, others can back it up.

Highlight #2 History of Git

Git is a system that is based on maximizing speed, leading a simple design, having strong support for non-linear development, being fully distributed, and able to handle large projects. This system was created by Linus Torvalds, the creator of Linux, a system that is similar to that of Git. Since its 2005 release, Git has grown exponentially due to its user friendly interface and capability to outperform its contestants.

Highlight #3 Getting a Git Repository

As of 2005, Git has been taking over the market as an efficient system, which makes it pretty crucial for developers to sign up for a Git Repository. There are two approaches to getting a Git repository: taking an existing project or directory and imports it into Git, and cloning an existing Git repository from another server.

Highlight #4 GitHub

GitHub is a web based git or version control repository and internet hosting service. Born in 2005, GitHub's user friendly interface has attracted a massive crowd of daily users. Users are able to do this through command-line interface and standard git commands.

Highlight #5 Basic Git Commands

There are a lot of amazing things you can accomplish through Git; however, they are impossible to achieve without knowing the functions of basic git commands. To get users started, some basic, yet important commands include: letting know git who "you" are (git config --global user.name "John Doe"), creating a new local depository (git init), commit (git commit -a), and push (git push origin master).
